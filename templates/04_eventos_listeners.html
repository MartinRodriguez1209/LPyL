🎯 Template — Eventos & Listeners

1. Formas de asignar eventos

// ── ❌ Forma 1: atributo HTML (evitar en código serio) ────────
// <button onclick="miFuncion()">

// ── ⚠️  Forma 2: propiedad on* (solo un listener por evento) ──
btn.onclick = function() { console.log('click'); };
btn.onclick = otraFuncion;  // sobreescribe la anterior

// ── ✅ Forma 3: addEventListener (RECOMENDADA) ────────────────
btn.addEventListener('click', function(event) {
  console.log('click!', event);
});

// Con arrow function
btn.addEventListener('click', (e) => {
  console.log(e.target);  // elemento que disparó el evento
});

// Con función nombrada (permite removeEventListener)
function handler(e) { console.log(e); }
btn.addEventListener('click', handler);
btn.removeEventListener('click', handler); // quitar listener

2. El objeto Event (e / event)

Esperando evento...
document.getElementById('btn').addEventListener('click', (e) => {

  // ── Información del evento ──────────────────────────────────
  e.type;           // 'click', 'keydown', 'submit', etc.
  e.timeStamp;      // milisegundos desde que cargó la página

  // ── El elemento ─────────────────────────────────────────────
  e.target;         // elemento que disparó el evento (donde se clickeó)
  e.currentTarget;  // elemento donde está el listener

  // ── Mouse ────────────────────────────────────────────────────
  e.clientX;  e.clientY;   // posición relativa a la ventana
  e.pageX;    e.pageY;     // posición relativa al documento
  e.button;                // 0=izq, 1=medio, 2=der

  // ── Teclado ──────────────────────────────────────────────────
  e.key;        // 'Enter', 'a', 'ArrowUp', etc.
  e.code;       // 'KeyA', 'Enter', 'Space' (físico)
  e.ctrlKey;    // true si Ctrl presionado
  e.shiftKey;   // true si Shift presionado
  e.altKey;     // true si Alt presionado

  // ── Controlar el comportamiento ──────────────────────────────
  e.preventDefault();  // cancela la acción por defecto
                        // (submit de form, seguir link, etc.)
  e.stopPropagation(); // detiene el bubbling
});

4. Eventos de Teclado

Esperando tecla...
Teclas especiales: EnterEscape ArrowUpArrowDown BackspaceTabSpace 
'keydown'   // tecla presionada (se repite si se mantiene) ← más usado
'keyup'     // tecla soltada
'keypress'  // ⚠️ deprecated, usar keydown

// Ejemplo: detectar Enter
input.addEventListener('keydown', (e) => {
  if (e.key === 'Enter') { enviar(); }
  if (e.key === 'Escape') { cancelar(); }
  if (e.ctrlKey && e.key === 's') { guardar(); } // Ctrl+S
});

5. Eventos de Formulario

Esperando...
'submit'   // cuando se envía el formulario
'change'   // cuando cambia el valor Y pierde foco
'input'    // CADA VEZ que cambia el valor ← tiempo real
'focus'    // cuando el campo recibe foco
'blur'     // cuando el campo pierde foco
'reset'    // cuando se resetea el formulario

// ── Ejemplo input en tiempo real ─────────────────────────────
input.addEventListener('input', (e) => {
  console.log('Valor actual:', e.target.value);
});

// ── Ejemplo submit ───────────────────────────────────────────
form.addEventListener('submit', (e) => {
  e.preventDefault();           // ← SIEMPRE para no recargar
  const valor = input.value;
  console.log('Enviado:', valor);
});

6. Delegación de eventos

Hacé clic en los items (incluido el que se agrega dinámicamente):

Item 1
Item 2
Item 3
Esperando clic...
// ── Problema: listener en cada hijo ──────────────────────────
items.forEach(item => item.addEventListener('click', fn));
// ❌ Los elementos creados DESPUÉS no tienen el listener

// ── Solución: delegación en el PADRE ─────────────────────────
const lista = document.getElementById('lista');
lista.addEventListener('click', (e) => {

  // Verificar que se clickeó el elemento correcto
  if (e.target.matches('div')) {
    console.log('Item clickeado:', e.target.dataset.id);
  }

  // O con closest (útil si hay elementos hijos dentro)
  const item = e.target.closest('[data-id]');
  if (item) { console.log(item.dataset.id); }
});
// ✅ Funciona para items que se agregan dinámicamente también

7. Bubbling & Capturing

// ── Bubbling (por defecto) ────────────────────────────────────
// El evento sube: hijo → padre → abuelo → document
// Si clickeás un <span> dentro de un <div>, ambos reciben el evento

hijo.addEventListener('click', (e) => {
  e.stopPropagation();  // detiene el bubbling hacia el padre
});

// ── Capturing (fase de captura, menos común) ──────────────────
// El evento baja: document → abuelo → padre → hijo
padre.addEventListener('click', handler, true);  // ← true activa capture

// ── Orden completo de un evento ──────────────────────────────
// 1. Captura (document → target)  [fase capture]
// 2. Target (el elemento clickeado) [fase target]
// 3. Bubbling (target → document) [fase bubble]
